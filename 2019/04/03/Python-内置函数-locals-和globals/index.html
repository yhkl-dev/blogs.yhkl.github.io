Python--内置函数 locals() 和globals()
event Posted on: 2019-04-03 event_available Edited on: 2021-10-30 folder In: Uncategorized visibility Views: 
1.名字空间

Python有两个内置函数locals()和globals()，这两个函数提供了基于字典的访问局部变量和全局变量的方式

在使用两个函数之前，需要了解Python记录变量的方式：名字空间

Python使用名字空间来记录变量的轨迹，名字空间是一个字典，可以像dictionary一样进行访问

  • 局部名字空间，每个函数都有自己的名字空间， 记录了函数的变量，包括函数的参数和局部定义的变量

  • 全局名字空间，每个模块拥有自己的名字空间，记录了模块的变量，包括函数、类、其他导入的模块、模块级的变量和常量

  • 内置名字空间，任何模块均可访问，存放内置的函数和异常

2.变量的访问顺序

当某行代码需要使用某个变量时，Python会到所有可用的名字空间去查找变量，查找顺序如下：

  • 局部名字空间，指当前函数或类的方法，如果函数定义了所需一个局部变量或者一个参数， Python将直接使用它并停止搜索
  • 全局名字空间，指当前模块，如果模块定义了所需要的变量、函数或类，Python将直接使用并停止搜索
  • 内置名字空间，对每个模块都是全局的，上述两个名字空间都找不到所需变量时，Python会在内置名字空间中寻找所需变量，如果依然找不到，会抛出There is no variable named 'variable'异常

名字空间在运行时可以直接访问，局部名字空间可以通过内置的locals函数来访问，全局名字空间可以通过内置的globals函数来访问。

locals和globals对名字空间的操作是等价的，但是globals的功能更为强大，模块的名字空间包含了模块级别的变量和常量，还包括了所有在模块中定义的函数和类，以及任何被导入到模块中的东西

3.from module import和import module的区别：
  • import module， 模块自身被导入，但是它保持着自己的名字空间，访问时需要使用module.function_name
  • from module import , 本质上是从另一个模块中将指定的函数和属性导入当前的名字空间，访问时直接使用function_name即可，不需引用该函数/类所在的模块名称
4. locals和globals的区别

locals是只读的，不可修改，globals可以修改

原因如下：

  • locals实际上没有返回局部名字空间，它返回的是局部明名字空间的深拷贝，对其进行修改，修改的是拷贝，对局部名字空间的值并无影响

  • globals返回的是实际的全局名字空间，而不是一个拷贝，这一点和locals的动作完全相反
